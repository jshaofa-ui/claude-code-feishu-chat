## 📋 问题诊断

### 问题 1：主聊天创建话题回复
**现象**: 每次在主聊天发送消息，Claude Code 回复都会创建新话题

**原因**: 
- 使用了 `+messages-reply --reply-in-thread` 命令
- 飞书私聊所有消息都有 `thread_id`，导致误判

**修复**: 
```javascript
// 修复前
const isThread = !!(msg.thread_id || msg.root_id || msg.parent_id);

// 修复后
const isThread = !!(msg.root_id);  // 只有话题回复才有 root_id
```

---

### 问题 2：话题内无法回复
**现象**: 在话题窗口发送消息，Claude Code 不回复

**原因**: 
- `+messages-reply` 命令使用了 `--chat-id` 参数（不支持）

**修复**: 
```javascript
// 修复前
cmd = `lark-cli im +messages-reply --chat-id "${chatId}" --message-id "${replyId}" ...`;

// 修复后
cmd = `lark-cli im +messages-reply --message-id "${replyId}" ...`;
```

---

### 问题 3：回复被拆分成多个话题
**现象**: Claude Code 回复被飞书自动拆分成多条消息

**原因**: 
- 回复太长（超过飞书限制）
- 包含特殊字符（反引号、引号等）导致 shell 语法错误

**修复**: 
```javascript
// 1. 限制回复长度
const escapedText = text.substring(0, 500);

// 2. 转义特殊字符
const escapedText = text
  .replace(/\\/g, '\\\\')
  .replace(/"/g, '\\"')
  .replace(/`/g, '\\`')
  .replace(/\$/g, '\\$');
```

---

### 问题 4：思考过程丢失
**现象**: Claude Code 的思考过程被过滤掉

**原因**: 
- 代码中清理了思考标记

**修复**: 
```javascript
// 保留完整回复（包括思考过程）
function callClaude(text) {
  const prompt = `请简洁回复（200 字内）：${text}`;
  // 获取完整回复，限制 500 字
}
```

---

### 问题 5：重复回复
**现象**: 同一条消息被多次回复

**原因**: 
- `processedMessages` 标记时机不对

**修复**: 
```javascript
// 立即标记为已处理，防止重复回复
processedMessages.add(msg.message_id);

// 然后才处理消息
const reply = await callClaude(text);
```

## 📝 经验总结

### 飞书 API 要点

1. **私聊消息都有 `thread_id`** - 不能用 `thread_id` 判断是否话题消息
2. **话题回复才有 `root_id`** - 正确的判断方式是检查 `msg.root_id`
3. **`+messages-reply` 不支持 `--chat-id`** - 只需 `--message-id`
4. **`+messages-send` 需要 `--chat-id`** - 发送新消息到指定聊天

### Shell 命令转义

```javascript
// 必须转义的特殊字符
const escapedText = text
  .replace(/\\/g, '\\\\')  // 反斜杠
  .replace(/"/g, '\\"')    // 双引号
  .replace(/`/g, '\\`')    // 反引号
  .replace(/\$/g, '\\$');  // 美元符号
```

### Claude 回复优化

```javascript
// 明确要求简洁回复
const prompt = `请简洁回复（200 字内）：${text}`;

// 限制总长度
reply = reply.trim().substring(0, 500);
```
