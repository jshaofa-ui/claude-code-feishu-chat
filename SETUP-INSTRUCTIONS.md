# 飞书-Claude Code 集成 - 配置说明

## 当前状态
系统已成功部署以下功能：
- WebSocket 长连接监听
- Claude Code 消息处理
- 消息去重和持久化

## 问题原因
目前无法收发消息的根本原因是飞书 API 的权限限制：
1. 机器人无法主动加入私聊窗口
2. 机器人无法向不在其中的聊天发送消息
3. 跨应用权限限制

## 解决方案
要使收发功能正常工作，请按以下步骤操作：

### 步骤 1: 创建与机器人的对话
您需要先与机器人建立对话：
1. 在飞书中搜索并找到机器人（通过 appId: cli_a91d6fb107f8dbd8）
2. 或者，请将机器人邀请到一个群聊中

### 步骤 2: 确认权限
确保机器人有权限接收和发送消息：
1. 在聊天中@机器人，确认它能收到消息
2. 检查机器人是否被禁言

### 步骤 3: 验证集成
一旦机器人加入了聊天，WebSocket 集成就会自动处理消息：
1. 在聊天中发送消息给机器人
2. 机器人将使用 Claude Code 处理消息
3. 机器人将回复处理结果

## 测试集成
要验证集成是否工作，请在机器人存在的聊天中发送消息：
"你好 claude"

## 技术细节
- WebSocket 长连接：正常运行
- Claude Code 处理：正常运行
- 消息去重：正常运行
- 权限问题：需要用户操作解决

## 故障排除
- 如果仍然没有响应，请确认机器人已加入您发送消息的聊天
- 检查飞书开发者后台的权限设置
- 确认 lark-cli 配置正确
